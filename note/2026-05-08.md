# 单击关闭蜂鸣器不灵敏问题分析

**日期**: 2026-05-08

## 问题描述

单击 KEY1 无法灵敏关闭蜂鸣器，需要等待约 1 秒后才能触发回调。

## 根本原因

问题定位在 `Key.c` 的 ISR 函数中：

```c
static void IRAM_ATTR key_isr_handler(void *arg)
{
    ...
    xTimerResetFromISR(ctx->timer, &higher_woken); // ← bug
    ...
}
```

`xTimerResetFromISR` 会把定时器重置为**当前周期**并重新倒计时。

### 问题时序

```
按下 → 去抖(20ms) → STATE_WAIT_RELEASE（定时器周期被改为 1000ms）
                                    ↓
松开手 → ISR 触发 → xTimerResetFromISR → 以当前周期(1000ms)重新开始倒计时！
                                    ↓
                         等足 1000ms 后定时器才回调
                                    ↓
                         level=1 → 才触发 KEY_EVENT_SINGLE_CLICK
```

**结论**：单击要等待最长 1000ms 后才能触发回调，蜂鸣器无法马上关闭。

### 修复方案

把 ISR 中的 `xTimerResetFromISR` 改为 `xTimerChangePeriodFromISR`，强制使用**短周期（消抖时间）**唤醒定时器：

```c
static void IRAM_ATTR key_isr_handler(void *arg)
{
    key_ctx_t *ctx = (key_ctx_t *)arg;
    BaseType_t higher_woken = pdFALSE;

    // 修复：不管当前定时器周期是多少，一律用消抖时间重新触发
    // 防止在 STATE_WAIT_RELEASE（周期=1000ms）时，松手的上升沿
    // 被 Reset 成继续等 1000ms 才回调的问题
    xTimerChangePeriodFromISR(ctx->timer, pdMS_TO_TICKS(KEY_DEBOUNCE_MS), &higher_woken);

    if (higher_woken) portYIELD_FROM_ISR();
}
```

### 修复后的时序

```
按下 → 去抖(20ms) → STATE_WAIT_RELEASE（周期改为 1000ms）
                                    ↓
松开手 → ISR 触发 → ChangePeriod(20ms) → 20ms 后回调
                                    ↓
                         level=1 → 立即触发 KEY_EVENT_SINGLE_CLICK ✅
                                    ↓
                         buzzer_notify_off_from_key() → 蜂鸣器马上关闭 ✅
```

## 关键点

- 状态机本身（`key_timer_cb`）逻辑不需要改动，因为各状态内部的 `xTimerChangePeriod` 调用本来就是正确的
- ISR 这里不能用 `xTimerResetFromISR`，否则会沿用可能长达 1000ms 的当前周期
- 应该用 `xTimerChangePeriodFromISR` 强制用消抖时间重新触发

## 经验教训

FreeRTOS 定时器 API 使用时：
- `xTimerResetFromISR()` 重置为当前周期
- `xTimerChangePeriodFromISR()` 改为指定周期

在 ISR 中修改定时器周期时，必须考虑当前周期的状态，避免使用错误的 API 导致意外的长延迟。
