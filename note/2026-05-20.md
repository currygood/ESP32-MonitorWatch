### ULP启动竞争

## 问题现象
主CPU读取到的`wakeup_reason`为0，但ULP应该已经设置过该值

## 核心原因分析

虽然ULP程序在调用`ulp_riscv_wakeup_main_processor()`之前修改了`wakeup_reason`，但主CPU读到0的根本原因在于：**ULP程序在主CPU启动期间重新运行了，并执行了`wakeup_reason = 0`这行代码**。

### 启动竞争时序
1. **ULP检测异常**：将`wakeup_reason`设为1，调用`ulp_riscv_wakeup_main_processor()`
2. **ULP执行完毕**：ULP程序运行结束
3. **触发重新运行**：ULP定时器配置下，或异常退出导致重启，ULP立即开始新一轮的`main`函数
4. **主CPU启动耗时**：主CPU从收到唤醒信号到运行`app_main`，需要200-500ms
5. **ULP抢跑清零**：在这几百毫秒内，ULP已经开始了第二轮运行，执行了`wakeup_reason = 0;`
6. **主CPU读取**：主CPU执行到`ESP_LOGE`时，读到的是被ULP第二轮运行清零后的值

### %ld格式符问题
在ESP-IDF的RISC-V(S2/S3)编译器中，`uint32_t`通常被定义为`unsigned int`，使用`%ld`（long int）可能导致打印时的内存解析不匹配。

## 解决方案

### 方案A：移除ULP内的初始化清零（推荐）
不要在ULP的`main`函数开头清零这个变量，生命周期由主CPU控制

```c
/* 共享变量 */
volatile uint32_t wakeup_reason = 0;

int main(void) {
    sensors_init();
    // 删除这行：不要在main开始时清零
    // wakeup_reason = 0;

    while(1) {
        // ... 采集代码 ...

        if (hr_abnormal_consecutive_count >= MAX_HRABNORMAL_NUM) {
            wakeup_reason = 1;
            ulp_riscv_wakeup_main_processor();
            // 唤醒后让ULP进入死循环，防止return后重启
            while(1) { ulp_riscv_delay_cycles(ULP_RISCV_CYCLES_PER_MS * 100); }
        }
    }
}
```

### 方案B：主CPU在入睡前负责复位
在`app_main`中，进入深度睡眠前手动清零

```c
if(received_cmd == KEY2_LONGPRESS_ENTER_SLEEP || received_cmd == LOW_POWER_ENTER_SLEEP)
{
    ESP_LOGI("MainSleep", "准备进入深度睡眠...");

    // ... 关闭OLED、WiFi等 ...

    // 【关键】启动ULP之前，先把共享变量清零
    ulp_wakeup_reason = 0;

    // 初始化并启动ULP
    init_ulp_program();

    ESP_ERROR_CHECK(esp_sleep_enable_ulp_wakeup());
    esp_deep_sleep_start();
}
```

### 方案C：更安全的打印方式
确保打印时类型匹配

```c
ESP_LOGE("MainWake", "🔔 ULP 唤醒！reason=%u\n", (unsigned int)ulp_wakeup_reason);
```

## 为什么ULP会反复运行？
在ESP-IDF中，调用`esp_sleep_enable_ulp_wakeup()`时，系统通常会配置ULP定时器。即使`main`函数有`while(1)`，如果ULP因某种原因（内存溢出、看门狗或代码执行完毕return）结束，硬件定时器会根据配置的时间间隔重新拉起ULP。

通过在ULP唤醒主CPU后添加`while(1) { halt; }`逻辑，可以确保ULP在主CPU接管期间保持静默。

## 总结建议
1. 删掉ULP代码`main`函数里的`wakeup_reason = 0;`
2. ULP唤醒主CPU后，不要`break`退出`main`，而是进入空死循环等待主CPU
3. 清零操作完全交给主CPU：主CPU唤醒后读取 → 处理逻辑 → 准备下一次入睡前清零
