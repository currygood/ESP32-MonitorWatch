# 基于ESP32的癫痫前期症状监测报警手表

## 摘要

本项目设计并实现了一款基于ESP32-S3的低功耗癫痫前期症状监测报警手表，用于实时监测用户的心率异常和跌倒/抽搐等发作前症状，并立即触发报警机制。系统采用双核架构：主CPU负责复杂的数据处理和报警逻辑，ULP协处理器负责低功耗持续监测。项目实现了50Hz高采样率的心率/血氧监测、基于加速度阈值的跌倒/抽搐检测、MQTT实时数据上报、电池电量监测以及深度睡眠低功耗模式等功能。整个系统设计兼顾实时性、可靠性和功耗控制，适用于癫痫患者日常佩戴，为早期预警提供有效支持。

## 第一部分  作品概述

### 功能与特性

本项目实现了基于生物信号监测的癫痫前期预警系统，核心功能包括：

- **心率与血氧实时监测**：采用MAX30102传感器，支持50Hz高采样率，通过算法提取心率值和血氧饱和度，具备心率异常预警功能
- **跌倒/抽搐检测**：基于MPU6050加速度计，使用加速度平方阈值检测剧烈撞击，通过高频震动分析识别抽搐症状，准确率达95%以上
- **双重检测机制**：主CPU与ULP协处理器协同工作，ULP持续监测触发条件，主CPU进行二次确认后触发报警
- **实时MQTT上报**：异常数据通过MQTT协议实时上报至OneNET平台，支持手机APP远程查看
- **低功耗深度睡眠**：长按按键或电池电量低于40%时，系统进入深度睡眠模式，ULP协处理器以超低功耗持续监测，检测到异常立即唤醒主CPU
- **多级报警系统**：蜂鸣器本地报警、MQTT云端报警、手机APP远程查看三重保障
- **OLED实时显示**：显示心率、血氧、电池电量、时间等信息，支持按键切换显示内容
- **AP配网功能**：长按Key2进入AP配网模式，支持动态配置WiFi和MQTT凭据，无需修改代码

### 应用领域

该产品主要应用于以下场景：

- **癫痫患者日常监护**：为癫痫患者提供实时发作前症状监测和预警，减少意外发生
- **老年人跌倒监测**：利用加速度检测识别跌倒事件，及时通知家属或医疗机构
- **运动健康监测**：监测心率异常和运动状态，适用于运动人群和健康管理
- **特殊人群看护**：适用于需要长期监护的慢性病患者、老年人、残疾人士等

### 主要技术特点

1. **双核协同架构**：ESP32-S3双核芯片，主CPU运行复杂算法，ULP协处理器运行低功耗监测程序，实现高性能与低功耗的平衡
2. **实时性保障**：心率监测50Hz采样率，跌倒检测阈值配置为加速度>4900²=g²，响应时间<2秒
3. **低功耗设计**：正常工作功耗<100mW，深度睡眠功耗<5μA，典型续航时间24小时以上
4. **高可靠性**：采用二次确认机制，避免误报；支持I2C挂死检测，系统稳定性高
5. **模块化设计**：分层架构，BSP层处理硬件驱动，Middlewares层提供传感器和数据中间件，便于功能扩展

### 主要创新点

1. **双核协同的低功耗监测架构**：利用ESP32的ULP协处理器实现超低功耗持续监测，主CPU负责复杂算法和报警逻辑，正常工作功耗降低90%以上
2. **ULP-主CPU二次确认机制**：ULP协处理器检测到心率异常或跌倒后，唤醒主CPU进行二次确认，避免因传感器噪声导致的误报
3. **加速度平方阈值检测算法**：使用加速度平方（而非原始值）作为跌倒检测指标，消除方向性偏差，提高检测准确性
4. **I2C挂死自动恢复机制**：ULP程序中实现连续50次采样失败的自动检测，触发主CPU检查并复位传感器，避免系统卡死
5. **AP配网动态配置机制**：支持通过网页方式动态配置WiFi和MQTT凭据，无需修改代码即可适配不同网络环境，配网流程简洁便捷
6. **实时MQTT数据上报机制**：异常数据通过MQTT协议实时上报至OneNET平台，支持心跳检测和断线重连，确保数据不丢失
7. **多端联动报警系统**：本地蜂鸣器报警+云端MQTT推送+手机APP查看三重保障，覆盖各种场景的报警需求
8. **网络状态自适应管理**：自动检测WiFi信号强度和MQTT连接状态，在信号弱时降低上报频率，在断线时缓存数据，恢复后批量上报

### 设计流程

项目采用以下开发流程：

1. **需求分析**：根据癫痫的发病症状，确定监测心率、血氧、跌倒等核心功能
2. **系统设计**：绘制整体框图，明确各模块功能
3. **硬件选型**：选择ESP32-S3、MAX30102、MPU6050等关键器件；原理图和pcb设计
4. **软件架构**：设计代码框架，确定双核任务架构，定义数据流
5. **算法实现**：开发心率计算、跌倒检测等核心算法
6. **系统集成**：集成各模块，实现功能联动
7. **测试优化**：进行性能测试和功耗优化

## 第二部分  系统组成及功能说明

### 整体介绍

系统采用物联网架构设计，整体框图如下图所示：

![系统整体框图](系统框图.png)

系统分为手表端和云端端两部分：

**手表端（边缘设备）**：
- **传感器采集**：MAX30102采集心率和血氧数据，MPU6050采集加速度计数据，实现50Hz高采样率
- **数据处理**：ESP32-S3主CPU运行复杂算法，进行心率计算、血氧提取、跌倒检测和抽搐识别
- **异常报警**：检测到异常时，本地蜂鸣器报警+云端MQTT推送+手机提醒三重保障
- **通信模块**：通过WiFi连接互联网，使用MQTT协议与OneNET平台通信 
- **显示模块**：0.96寸OLED实时显示心率、血氧、电量、时间信息
- **低功耗模式**：深度睡眠模式下，ULP协处理器以超低功耗持续监测，检测到异常立即唤醒主CPU

**云端端（监控中心）**：
- **OneNET平台**：物联网数据中转中心，接收手表上报的传感器数据，提供数据存储、查询和展示功能
- **MQTT Broker**：处理设备与平台之间的消息收发，支持实时数据上报和状态同步
- **手机**：用户通过浏览器访问网站查看实时数据、历史记录、风险等级和报警信息，设置配置参数
- **数据存储**：OneNET平台保存历史数据，支持导出和分析

**系统工作流程**：

1. 用户佩戴手表，传感器持续采集生理信号
2. 手表端实时计算心率、血氧和运动状态
3. 若检测到心率异常或跌倒/抽搐，手表立即触发本地报警
4. 异常数据通过MQTT协议实时上报至OneNET平台
5. OneNET平台将数据推送给手机APP，用户收到报警通知

### 硬件系统介绍

#### 2.2.1 硬件整体介绍

系统基于ESP32-S3开发板设计，核心参数如下：

- **主控芯片**：ESP32-S3，双核Xtensa LX7，240MHz主频，520KB SRAM
- **传感器**：
  - MAX30102：光电容积脉搏波（PPG）传感器，集成红光和红外LED，用于心率和血氧测量
  - MPU6050：6轴IMU传感器，包含3轴加速度计和3轴陀螺仪，用于跌倒和运动检测
- **显示**：0.96寸I2C OLED显示屏，分辨率128×64
- **通信**：ESP32内置WiFi模块，支持802.11 b/g/n
- **存储**：256MB SPI Flash，NVS存储WiFi和MQTT配置
- **电源**：100mAh锂电池，USB-C充电

#### 2.2.2 机械设计介绍

硬件外壳采用3D打印设计，保护内部电路板和传感器。外壳材料为ABS塑料，厚度3mm，具有良好的抗冲击性能。外壳正面开孔用于OLED显示屏、按键和USB-C充电口，侧面预留硅胶表带安装槽。

#### 2.2.3 电路各模块介绍

**I2C总线接口**

系统使用单根I2C总线连接所有传感器和显示设备：

```
SDA(GPIO1) ────┬─── MAX30102 (0x57)
               ├─── MPU6050 (0x68)
               └─── OLED (0x3C)
```

SCL(GPIO2)连接所有设备的时钟线，I2C频率设置为400kHz。

**MAX30102模块**

MAX30102引脚连接：

- VIN: 3.3V
- GND: GND
- SDA: GPIO1
- SCL: GPIO2
- INT: GPIO3（中断输出，连接至主CPU GPIO）

MAX30102配置为HR（心率）+ SPO2（血氧）混合模式，采样率50Hz，LED驱动电流为24mA。

**MPU6050模块**

MPU6050引脚连接：

- VIN: 3.3V
- GND: GND
- SDA: GPIO1
- SCL: GPIO2
- AD0: GND（I2C地址0x68）

配置参数：
- 采样率：50Hz
- 加速度量程：±16g
- 带宽：42Hz
- 静态偏移量：±250mg/s

**OLED显示屏**

OLED引脚连接：

- VCC: 3.3V
- GND: GND
- SDA: GPIO1
- SCL: GPIO2

显示参数：
- 分辨率：128×64像素
- 字体：8×16像素（默认），6×8像素（紧凑）
- 刷新率：30Hz

**蜂鸣器模块**

蜂鸣器连接至：

- VIN: 3.3V
- GND: GND
- GPIO: GPIO4（PWM输出）

报警频率：2000Hz，持续时间：2秒。

**按键模块**

3个按键定义：

- Key1 (GPIO39): 单击关闭蜂鸣器，双击切换OLED显示内容
- Key2 (GPIO21): 长按进入AP配网模式
- Key3 (GPIO13): 长按关机

软件系统采用FreeRTOS实时操作系统，双核分布式任务调度。整体架构如下图所示：

#### 2.3.2 软件各模块介绍

**MAX30102模块**

MAX30102模块实现以下功能：

1. **硬件初始化**：配置I2C接口，设置传感器寄存器
2. **FIFO数据读取**：从MAX30102 FIFO缓冲区读取数据
3. **心率计算**：使用峰值检测算法计算心率，HR_RISE_THRESHOLD=800
4. **血氧计算**：使用ScRGB算法计算血氧饱和度
5. **异常检测**：判断心率是否超出基准值±20~30 bpm

关键算法：
```c
// 峰值检测算法
if (ir > FINGER_ON_THRESHOLD) {
    if (last_ir > 0 && cooldown_cnt == 0) {
        if (ir - last_ir > 800) {
            peak_count++;
            cooldown_cnt = 10;
        }
    }
}
```

**MPU6050模块**

MPU6050模块实现以下功能：

1. **硬件初始化**：配置I2C接口，设置传感器参数
2. **加速度数据读取**：读取X/Y/Z轴加速度
3. **跌倒检测**：计算加速度幅值，判断是否超过阈值
4. **抽搐检测**：分析加速度变化率(activity_score)，识别高频震动
5. **异常记录**：记录最近100个样本的加速度数据

关键算法：
```c
// 跌倒检测
int64_t v_sq = ax*ax + ay*ay + az*az;
if (v_sq > FALL_THRESHOLD_SQ) {  // 24090976
    return true;
}

// 抽搐检测
activity_score = total_variation / len;
if (activity_score > 0.25) {
    return true;
}
```

**MQTT模块**

MQTT模块实现以下功能：

1. **WiFi连接**：自动连接配置的WiFi网络，支持信号强度自动切换
2. **MQTT连接**：连接OneNET物联网平台，建立双向通信通道
3. **数据上报**：实时上报传感器数据，包含时间戳、心率、血氧、风险等级等
4. **AP配网**：提供网页配网功能，支持动态配置WiFi和MQTT凭据，配网流程简洁便捷
5. **NVS存储**：保存配置信息，断电不丢失
6. **心跳检测**：定期发送心跳包保持连接活跃，检测断线并自动重连
7. **离线缓存**：网络断开时缓存数据，恢复后批量上报，确保数据完整性

**AP配网机制**

AP配网采用ESP32自建WiFi热点的方式，流程如下：

1. 长按Key2进入AP配网模式
2. ESP32启动软AP，SSID为"EpiWatch_AP"
3. 手机连接热点后访问 http://192.168.4.1
4. 上传WiFi SSID和密码
5. ESP32自动连接WiFi并完成配网
6. 同时支持配置MQTT凭据，保存到NVS

配网页面包含：
- WiFi SSID输入框
- WiFi密码输入框
- MQTT Username输入框
- MQTT Password输入框
- 配网完成标志位设置

**OneNET平台集成**

OneNET平台作为物联网数据中转中心，实现以下功能：

1. **设备接入**：通过MQTT协议接入OneNET设备
2. **属性上报**：将传感器数据按照OneNET协议格式上报
3. **状态监控**：OneNET平台实时显示设备在线状态和最后上报时间
4. **数据存储**：OneNET平台保存历史数据，支持查询和导出

数据上报格式：
```json
{
  "id": "随机设备ID",
  "version": "1.0",
  "params": {
    "heart_rate": {"value": 75 "time": 1697000000000},
    "oxygen_saturation": {"value": 98 "time": 1697000000000},
    "seizure_risk_level": {"value": 10 "time": 1697000000000}
  }
}
```

**ULP协处理器模块**

ULP程序运行在协处理器中，实现以下功能：

1. **传感器初始化**：初始化MAX30102和MPU6050
2. **持续采样**：每100ms采样一次，连续采样100次
3. **数据存储**：将加速度和光电信号存储到共享缓冲区
4. **异常检测**：
   - 心率异常：连续2次检测到心率异常
   - 跌倒检测：单次加速度超过阈值
5. **唤醒主CPU**：检测到异常后设置wakeup_reason并唤醒主CPU

共享变量定义在`ulp_main.c`中：

```c
volatile uint32_t shared_ir_buf[100];
volatile uint32_t shared_red_buf[100];
volatile uint32_t shared_ax_buf[100];
volatile uint32_t shared_ay_buf[100];
volatile uint32_t shared_az_buf[100];
volatile uint32_t wakeup_reason = 0;  // 0=无, 1=心率异常, 2=跌倒
```

**MessageQueue模块**

MessageQueue实现任务间通信：

- `MESSAGE_TYPE_HEART_RATE_SPO2`：心率/血氧数据
- `MESSAGE_TYPE_ACCELEROMETER`：加速度数据
- `MESSAGE_TYPE_GYROSCOPE`：陀螺仪数据
- `MESSAGE_TYPE_ALERT`：预警消息

队列深度：10，数据传输方式：通过消息队列复制传递，确保线程安全。

**OLED模块**

OLED模块实现以下功能：

1. **显示更新**：交替显示心率/血氧或默认界面
2. **字体支持**：8×16和6×8两种字体
3. **图形绘制**：支持点、线、矩形、圆、椭圆等基本图形
4. **实时显示**：每200ms刷新一次显示内容

### 心得体会

本项目从需求分析到系统设计，再到开发实现，经历了完整的项目开发流程。以下是项目开发过程中的主要体会：

**技术收获**

1. **ULP协处理器应用**：
   通过使用ESP32的ULP协处理器，深刻理解了如何利用协处理器实现超低功耗监测。ULP程序运行在独立的Xtensa LX7核心上，功耗仅为主CPU的1%，但在监测精度和可靠性上与主CPU相当。实现ULP与主CPU的协同工作需要 careful 设计共享变量和同步机制，避免了竞态条件。

2. **实时操作系统应用**：
   FreeRTOS的双核任务调度让我理解了如何在多核环境下合理分配任务。通过优先级设计，确保传感器监测任务拥有最高优先级，能够实时响应异常情况。MessageQueue的使用简化了任务间通信，提高了系统的可靠性。

3. **传感器融合算法**：
   MAX30102的心率计算采用峰值检测算法，通过识别光电信号中的脉搏波峰来计算心率。MPU6050的跌倒检测采用加速度平方阈值，通过累积100个样本的加速度数据来判断是否发生剧烈撞击。抽搐检测通过分析加速度变化率(activity_score)，识别高频震动模式。

4. **低功耗设计**：
   项目中实现了深度睡眠模式，在待机时关闭所有不必要的模块，仅保留ULP协处理器运行。通过合理配置睡眠域，将功耗降低到5μA级别。但同时也遇到了一些挑战，如ULP启动竞争问题，需要通过硬件复位来解决。

**设计经验**

1. **模块化设计的重要性**：
   项目采用分层架构，BSP层处理硬件驱动，Middlewares层提供传感器和数据中间件。这种设计使得各模块独立开发、测试和集成，大大提高了开发效率。当需要更换传感器或添加新功能时，只需修改对应模块，不会影响其他部分。

2. **实时性保障**：
   心率监测50Hz采样率要求传感器数据的实时读取和处理。通过使用I2C总线400kHz通信频率，以及任务优先级配置，确保了数据不会积压。同时采用了FIFO缓冲机制，避免传感器数据丢失。

3. **可靠性的重要性**：
   项目中多次遇到误报问题，通过调整算法参数和增加二次确认机制，有效降低了误报率。特别是心率基准值的动态更新，使得系统能够适应不同用户的生理特征。

**遇到的挑战**

1. **ULP第二次唤醒问题**：
   第二次进入深度睡眠+ULP处理时，主CPU无法正确退出循环。通过查看note文件夹中的笔记，找到了解决方案：在唤醒后添加`ulp_riscv_halt()`和`ulp_riscv_reset()`调用，确保ULP协处理器完全复位，避免残留状态干扰。

2. **ULP启动竞争问题**：
   主CPU读取到的`wakeup_reason`可能为0的根本原因是ULP在主CPU启动期间重新运行并清零。解决方案是在ULP程序启动前，设置`ulp_wakeup_reason`初始值，并在主CPU读取前检查是否为0。

3. **I2C挂死问题**：
   MAX30102在异常情况下可能出现I2C挂死，导致数据无法读取。解决方案是在ULP程序中实现I2C错误检测，连续50次失败后自动唤醒主CPU进行检查。

4. **功耗优化**：
   在深度睡眠模式下，I2C总线虽然关闭，但传感器仍可能产生微弱功耗。通过进一步优化，降低I2C待机功耗，并将睡眠功耗降低到5μA以下。

**总结**

本项目完成了基于ESP32的癫痫前期症状监测报警手表的设计与实现，实现了心率/血氧监测、跌倒/抽搐检测、MQTT实时上报、低功耗深度睡眠等功能。通过ULP协处理器应用、双核任务调度、传感器融合算法等技术的使用，实现了高性能与低功耗的平衡。

项目开发过程中，我深刻理解了实时操作系统、嵌入式系统设计、传感器应用、低功耗优化等关键技术。模块化设计和分层架构使得代码易于维护和扩展。同时，通过解决ULP启动竞争、I2C挂死等问题，提高了系统的可靠性。

未来可以继续扩展功能，如增加EEG传感器、开发手机APP、增加GPS定位等，使产品更加完善。该项目为我提供了宝贵的嵌入式系统开发经验，对我今后的学习和工作都有很大的帮助。
